Advanced options

Use this page when you want to edit the whole keyword table at once, back it up, or restore it on another browser.

Open the table tools

Open the side panel and expand Advanced options. These buttons control the same keyword data that powers chips, category colors, ON/OFF states, and page highlights.

  • View JSON opens the editor with your current table. Use it when you want to rename categories, reorder keywords, change colors, or make several edits faster than clicking chips one by one.
  • Export JSON downloads a backup file. Use it before large edits, or when you want to copy the same keyword table to another Chrome profile.
  • Import JSON reads a backup file and replaces the current table only after the file passes validation.
  • Save validates the text in the editor, stores it, redraws the side panel, and asks the current page to refresh its highlights.
  • Close hides the editor without saving the unsaved text.

JSON shape

The file is one object with a categories list. Each item in that list becomes one category card in the side panel.

{
  "categories": [
    {
      "name": "Hard Skills",
      "color": "#4f8cff",
      "enabled": true,
      "keywords": [
        "python",
        "java",
        "react",
        "aws"
      ]
    },
    {
      "name": "Red Flags",
      "color": "#ff6b6b",
      "enabled": true,
      "keywords": [
        "visa sponsorship",
        "security clearance",
        "u.s. person"
      ]
    }
  ]
}

Keep the structure exactly like this: curly braces for objects, square brackets for lists, double quotes around text, and commas between items.

Field-by-field meaning

categories is the full table. The order of this list is the order of category cards in the side panel.

name is the category title shown in the side panel, such as Hard Skills or Red Flags. Keep category names unique so Stack Highlighter can tell cards apart.

color controls the category dot, chip color, and highlight color on the page. Use a six-digit hex color such as #14b8a6.

enabled controls whether that category participates in highlighting. Set it to false to keep the keywords in your table while pausing that category.

keywords is the list of words or phrases to find on job pages. A keyword can be a single term like python or a phrase like visa sponsorship.

Common edits

To add keywords, put more strings inside a category's keywords list. The side panel will show them as chips after you click Save.

{
  "name": "Hard Skills",
  "color": "#4f8cff",
  "enabled": true,
  "keywords": [
    "python",
    "java",
    "react",
    "aws",
    "kubernetes"
  ]
}

To pause a category without deleting it, change enabled from true to false. This removes that category's highlights but keeps the data for later.

{
  "name": "Soft Skills",
  "color": "#f2b84b",
  "enabled": false,
  "keywords": [
    "communication",
    "teamwork"
  ]
}

To make a new category, copy one category object, paste it inside categories, then change its name, color, and keywords.

Validation rules

Stack Highlighter rejects invalid JSON, duplicate category names, missing fields, invalid colors, and empty keyword lists. That is intentional: a bad import should not overwrite a working table.

For safer editing, click Export JSON first, make your changes in View JSON, then click Save. If something feels wrong, import the backup you exported.
